# 目录递归检索

一种改进的 RAG 检索策略，通过分层递归方式精确定位目标上下文。

## 五步流程

1. **意图分析**：拆解用户查询，生成多个检索条件
2. **初始定位**：向量检索找到高匹配度的核心目录
3. **精细探索**：核心目录内二次检索，高分结果更新到候选集
4. **递归深入**：有子目录则逐层重复精细检索
5. **结果聚合**：返回最相关的完整上下文

## 核心思想

"先锁定高分目录，再精细化内容探索"——既找到语义匹配片段，又理解信息所在全局上下文，检索准确率大幅提升。

## 对比传统 RAG

| | 传统 RAG | 目录递归检索 |
|--|---------|------------|
| 检索方式 | 单一向量检索 | 目录引导 + 层层细化 |
| 上下文理解 | 碎片化 | 保留全局上下文 |
| 调试难度 | 难 | 可视化轨迹可追踪 |
