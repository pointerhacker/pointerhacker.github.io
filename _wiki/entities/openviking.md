# OpenViking

字节火山引擎推出的开源 AI Agent 框架（[GitHub](https://github.com/volcengine/OpenViking)），核心解决 Agent 上下文管理的痛点，21.7k Stars。

> 传统 RAG 的问题是：碎片化向量存储 + 上下文爆炸 + 检索效果差 + 不可调试。
> OpenViking 的思路：将 Agent 所有上下文映射为虚拟文件系统，用目录结构组织，用协议（`viking://`）寻址。

## 核心设计理念

将 Agent 所需的所有上下文（记忆、资源、技能）映射为**虚拟文件系统**，通过 `viking://` 协议赋予每个信息唯一 URI，让开发者像用 `ls`、`find` 命令一样管理智能体大脑。

## 文件系统管理范式

将 Agent 所有上下文整合为分层虚拟文件系统：

| 目录 | 内容 |
|------|------|
| `resources/` | 外部资源（文档、代码仓库、网页） |
| `user/` | 用户偏好、使用习惯 |
| `agent/` | 技能、指令、任务记忆 |

## 分层上下文加载（L0/L1/L2）

将所有信息自动处理为三级结构，按需加载：

| 层级 | 名称 | 内容 | 用途 |
|------|------|------|------|
| L0 | 摘要层 | 高度浓缩摘要 | 快速检索和相关性判断 |
| L1 | 概览层 | 核心信息和功能描述 | Agent 规划阶段决策 |
| L2 | 详情层 | 原始完整数据 | 需要深度阅读时 |

**解决的问题：** 海量上下文一次性塞入 prompt → Token 成本高、窗口超限。分层加载让 Agent 只调用当前需要的信息。

## 目录递归检索

一种改进的 RAG 检索策略，通过分层递归精确定位上下文，5步完成：

1. **意图分析**：拆解用户查询，生成多个检索条件
2. **初始定位**：向量检索找到高匹配度的核心目录
3. **精细探索**：核心目录内二次检索，高分结果更新到候选集
4. **递归深入**：有子目录则逐层重复精细检索
5. **结果聚合**：返回最相关的完整上下文

**对比传统 RAG：**

| | 传统 RAG | 目录递归检索 |
|--|---------|------------|
| 检索方式 | 单一向量检索 | 目录引导 + 层层细化 |
| 上下文理解 | 碎片化 | 保留全局上下文 |
| 调试难度 | 难 | 可视化轨迹可追踪 |

## 可视化检索轨迹

完整记录每次检索的目录浏览和文件定位轨迹。开发者能清晰看到 Agent 如何找到信息，出问题时可通过轨迹快速定位根因。

## 自动会话管理

每次会话结束后，自动分析任务执行结果和用户反馈，更新对应记忆目录：

- **更新用户记忆**：写作风格、编码习惯，让 Agent 响应更贴合用户
- **积累智能体经验**：从任务执行中提取工具使用技巧、操作流程

实现 Agent"越用越聪明"的自我进化。

## 技术栈

- Python 3.10+
- Go 1.22+（AGFS 组件）
- C++17 编译器
